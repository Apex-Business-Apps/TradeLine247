# Manual Installation (No ImageMagick Required)

This guide provides step-by-step instructions for installing Android launcher icons when ImageMagick is not available on your system.

## Prerequisites

- ✅ Source icon: `hi-res-icon-512.png` (512x512 PNG, ~22KB)
- ✅ Text editor or IDE
- ✅ Web browser for online icon generation

## Step 1: Generate Icons Online

Visit one of these free online icon generators:

### Option A: App Icon Co (Recommended)
```
URL: https://appicon.co/
```
1. Upload `hi-res-icon-512.png`
2. Select "Android" platform
3. Choose these sizes:
   - 48x48 (mdpi)
   - 72x72 (hdpi)
   - 96x96 (xhdpi)
   - 144x144 (xxhdpi)
   - 192x192 (xxxhdpi)
4. Generate and download ZIP file

### Option B: Easy App Icon
```
URL: https://easyappicon.com/
```
1. Upload `hi-res-icon-512.png`
2. Select "Android" from platform dropdown
3. Generate icons
4. Download all sizes as PNG

### Option C: Icon Kitchen
```
URL: https://icon.kitchen/
```
1. Upload `hi-res-icon-512.png`
2. Select "Android" platform
3. Generate and download

## Step 2: Extract and Organize Icons

After downloading, you should have these files:

```
ic_launcher_48x48.png    (mdpi)
ic_launcher_72x72.png    (hdpi)
ic_launcher_96x96.png    (xhdpi)
ic_launcher_144x144.png  (xxhdpi)
ic_launcher_192x192.png  (xxxhdpi)
```

Rename them to match Android naming convention:
- `ic_launcher_48x48.png` → `ic_launcher.png` (for mdpi folder)
- `ic_launcher_72x72.png` → `ic_launcher.png` (for hdpi folder)
- etc.

You also need round versions. Generate them by:
1. Using the same online tools (most generate both square and round)
2. Or manually create round versions by asking the tools to generate "round" variants
3. Or use image editing software to add rounded corners

## Step 3: Create Required Folders

Open PowerShell in your project root and run these commands to create the folder structure:

```powershell
# Create mipmap folders
New-Item -ItemType Directory -Path "android\app\src\main\res\mipmap-mdpi" -Force
New-Item -ItemType Directory -Path "android\app\src\main\res\mipmap-hdpi" -Force
New-Item -ItemType Directory -Path "android\app\src\main\res\mipmap-xhdpi" -Force
New-Item -ItemType Directory -Path "android\app\src\main\res\mipmap-xxhdpi" -Force
New-Item -ItemType Directory -Path "android\app\src\main\res\mipmap-xxxhdpi" -Force
New-Item -ItemType Directory -Path "android\app\src\main\res\mipmap-anydpi-v26" -Force
```

## Step 4: Copy Icons Manually

Assuming your generated icons are in a folder called `generated_icons` in your project root:

```powershell
# Copy icons to mipmap folders (adjust paths as needed)
Copy-Item "generated_icons\ic_launcher_48x48.png" "android\app\src\main\res\mipmap-mdpi\ic_launcher.png"
Copy-Item "generated_icons\ic_launcher_round_48x48.png" "android\app\src\main\res\mipmap-mdpi\ic_launcher_round.png"

Copy-Item "generated_icons\ic_launcher_72x72.png" "android\app\src\main\res\mipmap-hdpi\ic_launcher.png"
Copy-Item "generated_icons\ic_launcher_round_72x72.png" "android\app\src\main\res\mipmap-hdpi\ic_launcher_round.png"

Copy-Item "generated_icons\ic_launcher_96x96.png" "android\app\src\main\res\mipmap-xhdpi\ic_launcher.png"
Copy-Item "generated_icons\ic_launcher_round_96x96.png" "android\app\src\main\res\mipmap-xhdpi\ic_launcher_round.png"

Copy-Item "generated_icons\ic_launcher_144x144.png" "android\app\src\main\res\mipmap-xxhdpi\ic_launcher.png"
Copy-Item "generated_icons\ic_launcher_round_144x144.png" "android\app\src\main\res\mipmap-xxhdpi\ic_launcher_round.png"

Copy-Item "generated_icons\ic_launcher_192x192.png" "android\app\src\main\res\mipmap-xxxhdpi\ic_launcher.png"
Copy-Item "generated_icons\ic_launcher_round_192x192.png" "android\app\src\main\res\mipmap-xxxhdpi\ic_launcher_round.png"
```

## Step 5: Create Adaptive Icon XML Files

Create these two XML files in `android\app\src\main\res\mipmap-anydpi-v26\`:

### ic_launcher.xml
```xml
<?xml version="1.0" encoding="utf-8"?>
<!-- AUTO-GENERATED by manual installation -->
<!-- DO NOT MODIFY MANUALLY -->
<adaptive-icon xmlns:android="http://schemas.android.com/apk/res/android">
    <background android:drawable="@color/ic_launcher_background"/>
    <foreground android:drawable="@mipmap/ic_launcher_foreground"/>
</adaptive-icon>
```

### ic_launcher_round.xml
```xml
<?xml version="1.0" encoding="utf-8"?>
<!-- AUTO-GENERATED by manual installation -->
<!-- DO NOT MODIFY MANUALLY -->
<adaptive-icon xmlns:android="http://schemas.android.com/apk/res/android">
    <background android:drawable="@color/ic_launcher_background"/>
    <foreground android:drawable="@mipmap/ic_launcher_foreground"/>
</adaptive-icon>
```

## Step 6: Update colors.xml

Create or update `android\app\src\main\res\values\colors.xml`:

```xml
<?xml version="1.0" encoding="utf-8"?>
<resources>
    <color name="ic_launcher_background">#E85D2A</color>
</resources>
```

## Step 7: Apply Protection

Set read-only protection on all installed files:

```powershell
# Protect PNG files
Get-ChildItem "android\app\src\main\res\mipmap-*\*.png" | ForEach-Object {
    Set-ItemProperty -Path $_.FullName -Name IsReadOnly -Value $true
}

# Protect XML files
Get-ChildItem "android\app\src\main\res\mipmap-anydpi-v26\*.xml" | ForEach-Object {
    Set-ItemProperty -Path $_.FullName -Name IsReadOnly -Value $true
}
```

## Step 8: Verify Installation

Run the verification script:

```powershell
.\verify-icons.ps1
```

You should see:
```
STATUS: PERFECT ✓
All 10 icon files installed and protected
```

## Troubleshooting

### Icons don't appear in Android Studio
- Clean and rebuild project: Build → Clean Project, then Build → Rebuild Project
- Invalidate caches: File → Invalidate Caches / Restart

### Icons appear wrong size on device
- Check that files are in correct folders
- Verify file names match exactly: `ic_launcher.png`, `ic_launcher_round.png`
- Ensure no extra files in mipmap folders

### Background color not applied
- Verify `colors.xml` exists and contains the correct color value
- Check that adaptive icon XML references `@color/ic_launcher_background`

### Files are "read-only" and can't be modified
This is intentional protection. To remove protection temporarily:

```powershell
# Remove protection from all icon files
Get-ChildItem "android\app\src\main\res\mipmap-*\*" | ForEach-Object {
    Set-ItemProperty -Path $_.FullName -Name IsReadOnly -Value $false
}
```
